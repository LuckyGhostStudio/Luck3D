## 一、引擎简介

**Luck3D** 是一个基于 OpenGL 的 3D 游戏引擎，提供可视化编辑器用于场景搭建和调试。引擎采用 ECS（Entity-Component-System）架构，支持 PBR（基于物理的渲染）材质、多光源光照、场景序列化等功能。

**当前版本核心能力**：

| 能力 | 说明 |
|------|------|
| ?? PBR 渲染 | 基于物理的 Metallic-Roughness 工作流 |
| ?? 多光源 | 方向光 / 点光源 / 聚光灯，支持多光源叠加 |
| ?? ECS 架构 | 实体-组件系统，灵活的组件组合 |
| ??? 场景管理 | 新建 / 打开 / 保存场景（YAML 格式） |
| ??? 可视化编辑 | Transform 操控手柄、鼠标拾取、属性面板编辑 |
| ?? 内置图元 | 5 种基础几何体（立方体 / 平面 / 球体 / 圆柱体 / 胶囊体） |
| ???????? 父子层级 | 实体间的父子关系，支持拖拽重排 |

## 二、编辑器界面

### 2.1 整体布局

Luck3D 编辑器采用 **DockSpace** 布局，所有面板可自由拖拽、停靠、调整大小。默认布局包含三个主要面板：

```
┌──────────────────────────────────────────────────────────────┐
│                         菜单栏                                │
│  File  |  Window  |  Help                                    │
├──────────────┬───────────────────────┬───────────────────────┤
│              │                       │                       │
│  Hierarchy   │      Scene 视口       │     Inspector         │
│  （场景层级） │    （3D 场景渲染）     │    （属性检查器）      │
│              │                       │                       │
│  · Entity A  │                       │  Transform            │
│    · Child 1 │    [3D 渲染画面]       │    Position: x y z    │
│    · Child 2 │                       │    Rotation: x y z    │
│  · Entity B  │                       │    Scale:    x y z    │
│  · Light     │                       │                       │
│              │                       │  MeshFilter           │
│              │                       │    Primitive: Cube    │
│              │                       │                       │
│              │                       │  MeshRenderer         │
│              │                       │    Material: ...      │
│              │                       │                       │
└──────────────┴───────────────────────┴───────────────────────┘
```

### 2.2 菜单栏

#### File 菜单

| 菜单项 | 功能 |
|--------|------|
| **New** | 创建一个空白新场景 |
| **Open...** | 打开文件对话框，选择 `.luck3d` 场景文件加载 |
| **Save** | 保存当前场景到已有路径（首次保存会弹出另存为对话框） |
| **Save As...** | 弹出文件对话框，将当前场景保存到指定路径 |
| **Quit** | 退出编辑器 |

#### Window 菜单

| 菜单项 | 功能 |
|--------|------|
| **Hierarchy** | 显示 / 聚焦 Hierarchy 面板 |
| **Scene** | 显示 / 聚焦 Scene 视口面板 |
| **Inspector** | 显示 / 聚焦 Inspector 面板 |

#### Help 菜单

| 菜单项 | 功能 |
|--------|------|
| **About Luck3D** | 显示引擎信息（待完善） |

### 2.3 Scene 视口

Scene 视口是编辑器的核心区域，用于实时预览 3D 场景渲染结果。

**功能**：
- 实时渲染场景中的所有可见实体
- 编辑器相机控制（平移 / 旋转 / 缩放）
- 鼠标点击选中实体
- Transform 操控手柄（平移 / 旋转 / 缩放 Gizmo）

### 2.4 Hierarchy 面板

Hierarchy 面板以**树形结构**显示场景中的所有实体及其父子层级关系。

**功能**：
- 点击实体名称**选中**该实体
- **拖拽**实体到另一个实体上，设置为其子节点
- **拖拽**实体到空白区域，解除父子关系（变为根节点）
- **右键菜单**：
  - 创建空实体（Empty Entity）
  - 创建内置图元实体（Cube / Plane / Sphere / Cylinder / Capsule）
  - 创建光源实体（Directional Light / Point Light / Spot Light）
  - 删除选中实体
  - 为选中实体添加组件

### 2.5 Inspector 面板

Inspector 面板显示当前选中实体的所有组件属性，支持实时编辑。

**功能**：
- 显示并编辑实体名称
- 显示并编辑所有组件的属性（拖拽滑块、输入数值、颜色选择器、纹理选择等）
- 添加新组件（通过 "Add Component" 按钮）
- 删除组件（通过组件标题栏的删除按钮）
- 材质属性编辑（Shader 切换、颜色、数值、纹理等）

## 三、场景管理

### 3.1 创建新场景

通过 **File → New** 创建一个空白场景。新场景不包含任何实体，需要手动添加。

> ?? 创建新场景会**丢弃当前未保存的更改**，请先保存。

### 3.2 打开场景

通过 **File → Open...** 打开文件对话框，选择 `.luck3d` 格式的场景文件。编辑器会加载场景中的所有实体、组件、材质数据。

支持通过**命令行参数**直接打开场景文件：
```bash
Luck3DApp.exe path/to/scene.luck3d
```

### 3.3 保存场景

| 操作 | 说明 |
|------|------|
| **File → Save** | 保存到当前文件路径。如果是新场景（从未保存过），会自动弹出另存为对话框 |
| **File → Save As...** | 弹出文件对话框，选择保存路径和文件名 |

### 3.4 场景文件格式

场景文件使用 **YAML** 格式，扩展名为 `.luck3d`。文件内容人类可读，便于版本控制（Git 友好）。

场景文件中保存的数据包括：
- 场景名称
- 所有实体及其 UUID
- 每个实体的所有组件数据（Transform、MeshFilter、MeshRenderer、Light 等）
- 材质数据（Shader 名称、所有材质属性值、纹理路径）
- 父子层级关系

## 四、实体与组件

Luck3D 采用 **ECS（Entity-Component-System）** 架构。场景中的每个对象都是一个**实体（Entity）**，实体本身只是一个 ID，所有数据和行为通过**组件（Component）** 附加。

### 4.1 实体（Entity）

每个实体在创建时自动拥有以下**必备组件**：

| 组件 | 说明 |
|------|------|
| **ID** | 64 位唯一标识符（UUID），用于引用和序列化 |
| **Name** | 实体名称，显示在 Hierarchy 面板中 |
| **Transform** | 位置、旋转、缩放 |
| **Relationship** | 父子层级关系（父节点 UUID + 子节点 UUID 列表） |

这些组件**不可移除**，是每个实体的基础。

### 4.2 基础组件

| 组件 | 说明 | 可编辑属性 |
|------|------|-----------|
| **Transform** | 空间变换 | Position (x, y, z)、Rotation (x, y, z)、Scale (x, y, z) |
| **Name** | 实体名称 | Name（字符串） |

### 4.3 渲染组件

要让一个实体在场景中**可见**，需要同时添加以下两个组件：

| 组件 | 说明 | 可编辑属性 |
|------|------|-----------|
| **MeshFilter** | 指定实体的几何形状 | Primitive Type（图元类型选择） |
| **MeshRenderer** | 指定实体的外观（材质） | Materials（材质列表，每个 SubMesh 对应一个材质） |

> ?? **MeshFilter** 决定"形状是什么"，**MeshRenderer** 决定"看起来怎样"。两者缺一不可。

### 4.4 光源组件

光源组件用于在场景中添加光照。一个实体只能添加**一种**光源组件。光源的方向由实体的 **Transform 旋转**决定。

| 组件 | 说明 | 可编辑属性 |
|------|------|-----------|
| **DirectionalLight** | 平行光（如太阳光） | Color（颜色）、Intensity（强度） |
| **PointLight** | 点光源（向所有方向发光） | Color（颜色）、Intensity（强度）、Range（范围） |
| **SpotLight** | 聚光灯（锥形光束） | Color（颜色）、Intensity（强度）、Range（范围）、InnerCutoffAngle（内锥角）、OuterCutoffAngle（外锥角） |

## 五、Transform 变换

### 5.1 属性说明

| 属性 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| **Position** | vec3 | 实体在局部坐标系中的位置 | (0, 0, 0) |
| **Rotation** | vec3 | 实体的旋转角度（欧拉角，单位：弧度） | (0, 0, 0) |
| **Scale** | vec3 | 实体的缩放比例 | (1, 1, 1) |

**坐标系**：Luck3D 使用**右手坐标系**：
- **X 轴**：红色，向右
- **Y 轴**：绿色，向上
- **Z 轴**：蓝色，向前（屏幕外）

**旋转说明**：
- Inspector 中显示的是**欧拉角**（人类可读），内部使用**四元数**计算（避免万向锁）
- 两者通过 `SetRotationEuler()` / `SetRotation()` 自动同步

### 5.2 父子层级

实体之间可以建立**父子关系**：

- **子实体的 Transform 是相对于父实体的**（局部坐标系）
- 移动/旋转/缩放父实体时，子实体会跟随变换
- 世界坐标 = 父实体世界矩阵 × 子实体局部矩阵（递归计算）

**操作方式**：
- 在 Hierarchy 面板中**拖拽**实体到另一个实体上 → 设为子节点
- **拖拽**到空白区域 → 解除父子关系，变为根节点

### 5.3 Transform 操控手柄

在 Scene 视口中选中实体后，会显示 **Transform 操控手柄**（基于 ImGuizmo），支持三种操控模式：

| 模式 | 快捷键 | 说明 |
|------|--------|------|
| **平移（Translate）** | G | 沿 X / Y / Z 轴拖拽移动实体 |
| **旋转（Rotate）** | R | 绕 X / Y / Z 轴旋转实体 |
| **缩放（Scale）** | S | 沿 X / Y / Z 轴缩放实体 |

**辅助功能**：
- **Ctrl + 拖拽**：启用刻度捕捉（平移/缩放 0.5 间隔，旋转 5° 间隔）
- 支持**本地坐标系 / 世界坐标系**切换

## 七、材质系统

### 7.1 材质概述

材质（Material）定义了物体表面的外观。每个材质由一个 **Shader**（着色器程序）和一组**材质属性**（颜色、数值、纹理等）组成。

**核心特性**：
- **Shader 内省**：引擎自动分析 Shader 源码，发现所有可编辑的属性，无需手动注册
- **属性自动发现**：切换 Shader 后，Inspector 面板自动显示新 Shader 的所有属性
- **属性保留**：切换 Shader 时，同名同类型的属性值会自动保留
- **默认材质**：未指定材质的物体使用内置默认材质（白色 PBR）
- **错误材质**：Shader 编译失败时自动 fallback 到品红色错误材质

### 7.2 Standard 材质属性

引擎内置的 **Standard** Shader 实现了完整的 PBR（基于物理的渲染）工作流：

| 属性 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `u_Albedo` | Color (RGBA) | 基础颜色 | 白色 (1, 1, 1, 1) |
| `u_AlbedoMap` | Texture2D | 基础颜色贴图 | 白色纹理 |
| `u_Metallic` | Float [0, 1] | 金属度 | 0.0 |
| `u_MetallicMap` | Texture2D | 金属度贴图 | 白色纹理 |
| `u_Roughness` | Float [0, 1] | 粗糙度 | 0.5 |
| `u_RoughnessMap` | Texture2D | 粗糙度贴图 | 白色纹理 |
| `u_AO` | Float [0, 1] | 环境光遮蔽强度 | 1.0 |
| `u_AOMap` | Texture2D | 环境光遮蔽贴图 | 白色纹理 |
| `u_NormalMap` | Texture2D | 法线贴图 | 默认法线纹理 |
| `u_EmissionColor` | Color (RGB) | 自发光颜色 | 黑色 (0, 0, 0) |
| `u_EmissionMap` | Texture2D | 自发光贴图 | 黑色纹理 |

### 7.3 Shader 切换

在 Inspector 面板的 MeshRenderer 组件中，可以为每个材质**切换 Shader**：

1. 点击材质的 Shader 名称下拉框
2. 选择已加载的 Shader（如 Standard、InternalError 等）
3. 材质属性列表会自动更新为新 Shader 的属性

### 7.4 材质编辑

在 Inspector 面板中，材质属性根据类型提供不同的编辑控件：

| 属性类型 | 编辑控件 |
|----------|----------|
| Float | 拖拽滑块 |
| Vec2 / Vec3 | 多分量拖拽滑块 |
| Vec4 / Color | 颜色选择器（RGBA） |
| Texture2D | 纹理预览 + 文件选择 |

## 八、光照系统

Luck3D 使用 **PBR 光照模型**，支持三种光源类型，所有光源参与同一个光照计算。

### 8.1 方向光（Directional Light）

模拟**无限远的平行光源**（如太阳光）。光线方向由实体的 **Transform 旋转**决定，位置无关。

| 属性 | 说明 | 范围 |
|------|------|------|
| Color | 光照颜色 | RGB 颜色选择器 |
| Intensity | 光照强度 | 0.0 ~ ∞ |

**使用建议**：场景中通常只需要 1~2 个方向光作为主光源和补光。

### 8.2 点光源（Point Light）

从一个点向**所有方向**均匀发射光线，光照强度随距离衰减。

| 属性 | 说明 | 范围 |
|------|------|------|
| Color | 光照颜色 | RGB 颜色选择器 |
| Intensity | 光照强度 | 0.0 ~ ∞ |
| Range | 光照范围（超出范围光照为零） | 0.0 ~ ∞ |

**使用建议**：适合模拟灯泡、火把、爆炸等局部光源。

### 8.3 聚光灯（Spot Light）

从一个点沿特定方向发射**锥形光束**，光照在内锥角和外锥角之间平滑衰减。

| 属性 | 说明 | 范围 |
|------|------|------|
| Color | 光照颜色 | RGB 颜色选择器 |
| Intensity | 光照强度 | 0.0 ~ ∞ |
| Range | 光照范围 | 0.0 ~ ∞ |
| InnerCutoffAngle | 内锥角（全亮区域边界，单位：度） | 0° ~ OuterCutoffAngle |
| OuterCutoffAngle | 外锥角（光照完全截止边界，单位：度） | InnerCutoffAngle ~ 90° |

**内外锥角说明**：
- **内锥角以内**：光照强度为 100%
- **内锥角 ~ 外锥角之间**：光照强度平滑衰减到 0%
- **外锥角以外**：无光照

**使用建议**：适合模拟手电筒、舞台灯、车灯等定向光源。

### 8.4 光源数量限制

| 光源类型 | 最大数量 |
|----------|----------|
| 方向光 | 4 |
| 点光源 | 8 |
| 聚光灯 | 4 |

> ?? 超出数量限制的光源将被忽略（不参与光照计算）。

## 九、编辑器相机

Scene 视口中的相机是**编辑器相机**，采用**轨道相机**模型（围绕焦点旋转），用于在编辑模式下浏览场景。

### 相机操作

| 操作 | 方式 | 说明 |
|------|------|------|
| **旋转** | Alt + 鼠标左键拖拽 | 围绕焦点旋转视角 |
| **平移** | Alt + 鼠标中键拖拽 | 平移相机和焦点 |
| **缩放** | 鼠标滚轮 | 拉近 / 拉远相机与焦点的距离 |

### 相机参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| FOV | 45° | 垂直视场角 |
| Near Clip | 0.01 | 近裁剪面距离 |
| Far Clip | 1000.0 | 远裁剪面距离 |
| Distance | 5.0 | 相机到焦点的初始距离 |

## 十、鼠标拾取

在 Scene 视口中**点击**物体即可选中对应的实体。

**工作原理**：引擎使用 GPU 加速的 Entity ID 缓冲区技术，精确到像素级别的拾取。

**操作**：
- **左键点击**物体 → 选中该实体，Hierarchy 面板高亮，Inspector 面板显示其属性

## 十二、当前限制与未来计划

### 当前限制

| 限制 | 说明 |
|------|------|
| **无模型导入** | 目前只能使用 5 种内置图元，不支持导入 .obj / .fbx / .gltf 等外部模型 |
| **无阴影** | 光源不会产生阴影效果 |
| **无 HDR** | 高光区域可能被截断，色调表现不够自然 |
| **无后处理** | 不支持 Bloom、FXAA、色调调整等后处理效果 |
| **无 Gizmo 可视化** | 场景中没有网格线、灯光方向指示器、坐标轴指示器等辅助显示 |
| **无渲染排序** | 透明物体可能渲染顺序不正确 |
| **无 Undo/Redo** | 编辑操作不支持撤销/重做 |
| **无 Play Mode** | 不支持运行时模式（场景播放/暂停/停止） |
| **单视口** | 只支持一个 Scene 视口 |

### 未来计划

以下功能已有详细设计文档，将按优先级逐步实施：

| 优先级 | 功能 | 说明 |
|--------|------|------|
| ?? 最高 | 渲染排序 + 延迟提交 | 优化渲染性能，为后续功能打基础 |
| ?? 高 | Gizmo 渲染系统 | 网格线、灯光可视化、坐标轴指示器 |
| ?? 高 | 模型导入 | 支持 .obj / .fbx / .gltf 格式 |
| ?? 高 | 阴影系统 | 方向光 Shadow Map + PCF 软阴影 |
| ?? 高 | HDR + Tonemapping | HDR 渲染 + ACES 色调映射 |
| ?? 中 | 后处理框架 | Bloom、FXAA 等效果 |
| ?? 中 | 渲染状态控制 | Per-Material 的 ZWrite / CullMode / BlendMode |
| ?? 远期 | 多 Pass 渲染 | 灵活的渲染管线架构 |
| ?? 远期 | 选中高亮描边 | 后处理描边效果 |
